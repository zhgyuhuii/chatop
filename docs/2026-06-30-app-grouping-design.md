# 已安装应用 — 分组功能设计（2026-06-30）

## 1. 目标与范围

在应用管理弹窗的「**本机已安装**」(`__installed__`) 视图里，引入手机桌面式的**分组（文件夹）**能力：分组卡片与应用卡片在同一可无限下滑的网格中**混排**，支持任意拖拽排序、拖拽归组、分组展开、增删改、多选批量入组。

**作用范围**：仅应用管理弹窗内的已安装应用视图。**不改 XFCE 真实桌面、不动 .desktop 文件布局**（风险最小）。左侧控制栏仍是「应用管理」入口按钮，点开即此视图。

**不做（YAGNI）**：撤销栈、分组配色/自定义封面图标、iOS 抖动编辑态。

## 2. 交互设计

### 2.1 网格（顶层）
- 一个统一有序序列，元素分两类卡片：**应用卡** 与 **分组卡**，混排，可无限下滑。
- 拖拽任意卡片换位，顺序持久化。
- 拖到列表顶/底边缘时自动滚动。
- 末尾常驻一个「**+ 新建分组**」卡（显式建空组）。
- 右上角一个「**多选**」开关，进入多选批量模式（见 2.5）。

### 2.2 应用卡
- 图标 + 名称 + 「**详情**」按钮（由原「打开」改名）。
- **点图标/卡片本体 = 直接启动应用**（`POST /apps/run`，按 .desktop key）。
- 点「详情」：catalog 里有对应条目 → 进详情页；没有 → 仅显示基本信息/置灰。
- 拖一个应用卡**叠到另一个应用卡**上（重叠 >0.5s 高亮）→ 松手**自动建组**，两者入组，默认名「新建分组」。
- 拖一个应用卡**叠到分组卡**上 → 入该组。

### 2.3 分组卡
- 封面 = 组内**前 4 个**应用图标 2×2 拼贴；不足 4 个按 1/2/3 自适应；随组内增删/排序实时重算。
- 名称 + 右下角数量角标「N」。
- **点分组卡 = 居中浮层展开**（见 2.4）。
- 右键 / 长按菜单：重命名、解散。

### 2.4 分组展开浮层（模态内居中浮层）
- 半透明遮罩 + 居中卡片，盖在应用弹窗之上。
- 顶部：**可点击编辑的分组标题** + 关闭 ×。
- 主体：组内应用网格，点应用 = 启动；可拖拽排序（影响封面前 4）。
- 把应用**拖出浮层** → 移出该组，回到顶层网格。
- 「**+ 添加应用**」按钮 → 弹未归组应用列表（带勾选）→ 批量加入本组。
- 关闭：点遮罩 / ×。
- 组内拖到只剩 ≤1 个 → **自动解散**，剩余应用回顶层。

### 2.5 多选批量
- 顶层「多选」开关开启后，应用卡出现勾选框，可多选。
- 选中若干后底部出现操作条：「移入分组…」→ 选已有分组或新建 → 批量归入。
- 退出多选模式清空选择。

### 2.6 系统应用自动分组
- 启动加载时，`source=system` 的应用自动归入「**系统应用**」组（`auto:true`）。
- 用户可把系统应用拖出；拖出的 key 记入 `pulled_out_system`，**之后不再自动塞回**。
- 「系统应用」组与普通组操作一致，可重命名、可解散。

### 2.7 搜索
- 搜索框输入时**忽略分组**，跨所有分组 + 顶层平铺匹配结果（同 iOS Spotlight）。

### 2.8 边界
- 新装应用：出现在顶层网格**末尾**，未归组。
- 卸载应用：从所在组移除；若该组因此剩 ≤1 个 → 解散。

### 2.9 卸载（每个用户安装的应用都可卸载）
- **凡用户安装的应用都带卸载入口**，不再要求能对应到 catalog。卸载命令由后端从 .desktop **安装特征自动推断**：
  - exec 含 `proot-apps run <id>` → `proot-apps remove <id>`
  - `~/.local/share/applications/chatop-<id>.desktop` 且存在 `~/Applications/<id>` → `gui-uninstall.sh <id>`
  - 能对应 catalog 且其有 `remove` → 用 catalog 的 remove
- **系统应用（`source=system`，apt 装、需 root）不显示卸载按钮**：uid 1000 无 sudo，且卸载桌面组件会破坏桌面（已与用户确认此边界）。
- 卸载入口位置：已安装应用「详情」页里的「卸载」按钮（与 catalog 应用详情一致）。卸载成功后自动从分组/网格移除并按 2.8 收敛。

## 3. 数据与后端

### 3.1 持久化
- 文件：`~/.local/share/chatop/groups.json`（持久卷，重建不丢；XDG_DATA_HOME 已修正回卷路径）。
- 由 app-manager 后端读写。应用唯一标识 = `.desktop` 文件名 key（与 `installed_apps()` 一致）。

### 3.2 结构
```json
{
  "version": 1,
  "items": [
    {"type": "app", "key": "google-chrome.desktop"},
    {"type": "group", "id": "g1", "name": "办公",
     "apps": ["wps-office-wps-pa.desktop", "wechat-pa.desktop"]},
    {"type": "group", "id": "sys", "name": "系统应用", "auto": true,
     "apps": ["org.xfce.thunar.desktop"]}
  ],
  "pulled_out_system": ["xterm.desktop"]
}
```

### 3.3 接口
- `GET /apps/groups` → 返回 `groups.json`（若不存在则返回空布局）。
- `PUT /apps/groups` → 前端拖拽/编辑后整体保存布局（覆盖写）。
- `POST /apps/uninstall {key}` → 后端按 key 推断卸载命令并执行（带日志，复用 `/apps/logs?id=<key>` 轮询）。系统应用 / 无法推断者返回 400。
- `installed_apps()` 每项新增 `removable`（bool）：仅 `source=user` 且能推断出卸载命令时为 true。
- 均经 Caddy cookie 网关鉴权（与现有 `/apps/*` 一致）。

### 3.4 合并/对账（后端或前端加载时）
将 `installed_apps()` 全集与 `groups.json` 对账：
1. groups.json 里已不存在的应用 key → 从 items/组内剔除。
2. 不在任何组、也不在顶层 items 的新应用：若 `source=system` 且不在 `pulled_out_system` → 进「系统应用」组；否则 → 顶层末尾。
3. 空组（apps 为空）→ 删除。

## 4. 前端落点

- `novnc-src/app/ui.js` 的 `ChatopApps`：
  - state 增 `groups`、`selectMode`、`selected`、`openGroupId`。
  - `refresh()` 增 `GET /apps/groups`。
  - 重写 `__installed__` 分支：渲染混排网格（分组卡 + 应用卡）。
  - 新增：拖拽（HTML5 draggable）、建组/归组/出组、分组浮层、多选批量、保存（`PUT /apps/groups`，防抖）。
- `novnc-src/app/styles/base.css`：分组卡封面拼贴、浮层、多选态、拖拽态样式（含 `body.chatop-theme-light` 适配）。
- `app-manager/app_manager.py`：`GET/PUT /apps/groups` + 对账逻辑 + groups.json 读写（原子写）。

## 5. 实施阶段（后续 plan 细化）

1. 后端：groups.json 读写 + `GET/PUT /apps/groups` + 对账 + 系统应用自动分组。
2. 前端：混排网格渲染（分组卡封面拼贴、应用卡「详情」按钮、点击启动）。
3. 前端：拖拽（排序、应用→组、叠加建组、拖出）+ 保存防抖。
4. 前端：分组居中浮层（标题编辑、+添加应用、组内排序/拖出、自动解散）。
5. 前端：多选批量入组。
6. 搜索忽略分组 + 边界（新装落位、卸载收敛）+ 浅色主题适配 + 联调。

## 6. 验收要点
- 拖应用叠应用建组；拖应用入组/出组；空组自动解散。
- 分组封面随内容实时 2×2 拼贴 + 数量角标。
- 点应用图标=启动；「详情」=详情页。
- 系统应用自动归组且可拖出后不回流。
- 多选批量入组。
- 搜索跨组平铺。
- 重建容器后分组保持（groups.json 在卷）。
- 每个用户安装的应用（proot/AppImage/catalog）详情页都有「卸载」按钮且能真卸；系统应用无卸载按钮。
